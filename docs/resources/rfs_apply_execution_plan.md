# huaweicloud_rfs_apply_execution_plan

Manages a resource to apply execution plan within HuaweiCloud.

-> This resource is a one-time action resource used to trigger stack execution plan. Deleting this resource will
  not cancel the execution plan operation, but will only remove resource information from the tf state file.

## Example Usage

```hcl
variable "stack_name" {}
variable "execution_plan_name" {}

resource "huaweicloud_rfs_apply_execution_plan" "test" {
  stack_name          = var.stack_name
  execution_plan_name = var.execution_plan_name
}
```

## Argument Reference

The following arguments are supported:

* `region` - (Optional, String, ForceNew) Specifies the region in which to create the resource.
  If omitted, the provider-level region will be used.
  Changing this parameter will create a new resource.

* `stack_name` - (Required, String, NonUpdatable) Specifies the name of the stack to apply execution plan.

* `execution_plan_name` - (Required, String, NonUpdatable) Specifies the name of the execution plan to apply.

* `execution_plan_id` - (Optional, String) Specifies the unique ID of the execution plan.
  This ID is generated by Resource Formation Service when creating the execution plan and is a UUID.
  Using the ID ensures strong matching to avoid operating on the wrong execution plan when teammates delete and recreate
  execution plans with the same name.

* `stack_id` - (Optional, String) Specifies the unique ID of the stack.
  This ID is generated by Resource Formation Service when creating the resource stack and is a UUID.
  Using the ID ensures strong matching to avoid operating on the wrong stack when teammates delete and recreate
  stacks with the same name.

## Attributes Reference

In addition to all arguments above, the following attributes are exported:

* `id` - The resource ID, which is the same as the deployment ID.

## Timeouts

This resource provides the following timeouts configuration options:

* `create` - Default is 10 minutes.
